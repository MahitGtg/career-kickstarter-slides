Admin → Order Book

How instruments are created in the exchange

→ Admin authenticates directly with OB → CreateInstrumentRequest → OB validates + responds → OnOrderBookCreated → Gateway → all clients

Part 01 · Admin

Create Instrument

Admin → Order Book — direct TCP, separate port

{
  request_id:  12345,
  instrument: {
    symbol:      "AAPL",
    description: "Apple Inc.",
    currency:    "USD",
    multiplier:  1.0
  },
  tick_size: 0.01
}
01 Admin authenticates with OB
credentials never touch Gateway
02 OB validates: symbol uniqueness · tick_size > 0
03 Assigns unique order_book_id · creates book
04 Replies CreateInstrumentResponse directly
Gateway never sees this exchange

Part 01 · Instrument Delivery

OB → Gateway → All Clients

OnOrderBookCreated — the instrument is now live

Order Book → Gateway

Sends OnOrderBookCreated { order_book_id, tick_size, created_timestamp }

Gateway processes

price_depth_book.on_order_book_created(order_book_id)

instruments["AAPL"] = { ob_id: 1001, multiplier: 1.0 }

Gateway → All Clients

Broadcasts same OnOrderBookCreated message to every connected client

Clients update instrument list — can now subscribe and trade
