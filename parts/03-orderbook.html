Part 03 · Insert Flow

Insert — Three Phases

Every incoming order passes through validate → match → rest

① Validate

reject if qty <= 0
reject if price <= 0
reject if price % tick_size != 0

Fail → InsertFailure

→

② Match

while opposing side crosses:
    fill against best resting order
    emit BookTrade

Trades generated

→

③ Rest

if remaining > 0:
    add order to book
    enqueue at price level

remaining may be 0

Returns InsertSuccess: order_id · timestamp · trades[] · traded_quantity · resting_quantity

Part 03 · Insert — Matching

Match Buy — Sweep the Ask Side

Walk ask levels low → high while best_ask ≤ limit price

Incoming BUY

Price: 101.00   Qty: 400

while remaining > 0 and best_ask <= limit:
    fill against front of ask queue
    reduce or remove resting order

Step 1   best ask = 100.75 ≤ 101.00

Fill 300 at 100.75 → order #5 fully filled

remaining: 400 → 100

Step 2   best ask = 101.00 ≤ 101.00

Fill 100 at 101.00 → order #3 partially filled

remaining: 100 → 0

Part 03 · Insert — Resting & Reducing

Resting & Fills

Unfilled quantity joins the book · each fill decrements the passive order

Resting — when remaining > 0

create RestingOrder(id, side, price, qty)
store in orders dict
append to back of price level queue
update aggregate level quantity
New level?   Create queue, insert price in sorted list
Existing level?   Append to back of queue (FIFO)

Reducing — on each fill

new_remaining = old_remaining - fill_qty
if new_remaining == 0:
    remove order, pop from queue front
else:
    update order with new remaining

Part 03 · Cancel

Cancel — Remove a Resting Order

Cancel order #2 (BUY 200 @ 100.50) from the bid book

lookup order by id
if not found → CancelFailure

remove from price level queue
subtract qty from level total
if queue empty → delete price level
delete from orders dict

return CancelSuccess(remaining_qty)
vs matching:   cancel can remove from any position, not just front
cleanup:   if queue empties, price level is removed from sorted list

Part 03 · Internal Structures

Book Data Structures

Sorted price lists · FIFO queues per level · aggregate quantities

orders dict

order_id → RestingOrder

All live orders

O(1) lookup for cancel & matching

Sorted price levels

Bids: highest first
Asks: lowest first

O(log n) insert via bisect

O(1) access to best price

FIFO queues

price → deque[order_id]

One queue per price level

Front = highest time priority

Level quantities

price → total_qty

Sum of resting qty at each price. Updated on insert, fill, and cancel. Removed when zero.

ID allocation

Monotonic counters for order_id & trade_id

Each insert consumes an order_id. Each fill consumes a trade_id. Simple, deterministic.
